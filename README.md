# kirti-qa-exploratory-testing-portfolio

## Overview

This repository showcases independently identified bugs discovered during exploratory testing of publicly available Android applications from the Google Play Store.

The project focuses on demonstrating practical software testing skills including:
- Functional Testing
- Exploratory Testing
- UI Validation
- User Workflow Analysis
- Structured Bug Reporting

The purpose of this repository is to maintain a professional portfolio of real-world testing scenarios, reproducible defects, and detailed bug documentation created during independent QA testing activities.

## Featured Bug Report

### Application Tested
Rapido

### Bug Title
Liked location is not getting unliked from the search section.

### Bug Description
When a user taps the Like/Heart icon again from the search section to remove a saved location, the location does not get unliked. The UI continues displaying the location in the liked state even after the action is performed. This issue affects the expected toggle behaviour of the save/like functionality.
